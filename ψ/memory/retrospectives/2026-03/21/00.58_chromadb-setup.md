## Honest Feedback

**Friction 1: ChromaDB API breaking change** — ฉัน assume ว่า `list_collections` จะคืน list of strings (ตาม docs เก่า) แต่ v1.5.5 คืน Collection objects ทำให้ต้องแก้ 2 รอบ ครั้งแรกเปลี่ยนแค่ variable name ไม่ได้ช่วย ควร check type ก่อนเลยตั้งแต่แรก

**Friction 2: Windows cp1252 encoding** — Hieroglyphic unicode (𓂀) ไม่รองรับใน default Windows console encoding ต้อง set env var ทุกครั้ง ควรมี wrapper script หรือ set ใน project-level config

**Friction 3: No test verification** — ยังไม่ได้ทดสอบ add/query documents จริงๆ แค่ verify ว่า collections ถูกสร้าง ควรมี smoke test ที่ add document แล้ว query กลับมาด้วย

## Lessons Learned

- ChromaDB v1.5.5 `list_collections` returns Collection objects, not strings — iterate directly
- Windows Python scripts ต้อง handle encoding: `PYTHONIOENCODING=utf-8` หรือใช้ `chcp 65001`
- Persistent ChromaDB path ควรอยู่ใน brain structure (`ψ/memory/`) เพื่อให้สอดคล้องกับ architecture

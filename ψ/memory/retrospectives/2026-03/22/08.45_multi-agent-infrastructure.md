## Honest Feedback

**Friction 1: Bun statSync caching** — ใช้เวลากว่า 30 นาทีไล่หา bug ที่ FeedTailer ไม่ detect file changes ทั้งที่ feed-watcher เขียน feed.log จริง สาเหตุคือ Bun cache `statSync`, `fstatSync`, และ `Bun.file().size` ทั้งหมด ต้องแก้ด้วย `openSync` + `fstatSync` ใน poll loop ซึ่งเป็น workaround ไม่ใช่ solution ที่สวย ควรจะมี `fs.watch` หรือ inotify แทน polling

**Friction 2: Deploy cycle ช้า** — ทุกครั้งที่แก้ code ต้อง: build on Windows → copy to Ubuntu → kill server → start server 4 steps ที่ทำซ้ำ 10+ ครั้งใน session นี้ ควรมี hot-reload script หรือ dev mode ที่ auto-deploy

**Friction 3: maw-js ออกแบบมาสำหรับ Linux/macOS** — หลาย assumptions ไม่ตรงกับ Windows+WSL setup เช่น default path `/home/nat`, `getPaneCommand` regex, chat view hardcode "nat" เป็น human ต้องแก้หลายจุดเพื่อ adapt ให้ทำงานกับ setup ของพี่พง

## Lessons Learned

1. **Bun caches file metadata aggressively** — `statSync`, `fstatSync`, `Bun.file().size` ทุกตัว cache ใน WSL2 ต้องใช้ `openSync` + `fstatSync` ทุก poll cycle
2. **Cross-OS dev (Windows↔WSL2) มี friction สูง** — ย้ายทุกอย่างไป Ubuntu ลด complexity ลงมาก
3. **maw-js feed system** ต้องมี 3 ชิ้น: feed-watcher (JSONL→feed.log), FeedTailer (feed.log→WebSocket), frontend (WebSocket→UI status)
4. **Multi-agent needs shared state** — shared-threads folder + feed.log + maw-log.jsonl เป็น glue ที่ทำให้ agents เห็นกัน

## Honest Feedback

**Friction 1: Windows vs WSL tmux แยกกัน** — เสียเวลา 10+ นาทีพยายาม wake ผ่าน Windows tmux ทั้งที่ server จริงใช้ WSL tmux ควรเช็ค `wsl tmux list-sessions` ก่อนเสมอเมื่อเห็น maw API ตอบกลับ แต่ local tmux ว่าง นี่คือ pattern ที่ควรจำ: ถ้า API works แต่ local tmux ไม่เห็น → tmux อยู่ที่อื่น (WSL)

**Friction 2: Server path ไม่ตรงกับ repo** — maw server ทำงานจาก `/tmp/maw-js-server` ไม่ใช่จาก repo ตรง ทำให้ rebuild office แล้วไม่เห็นผล ต้อง copy ไฟล์ไปเอง ควรมี deploy script หรือ symlink ที่ sync อัตโนมัติ

**Friction 3: Duplicate import ซ่อนอยู่** — `describeActivity` import ซ้ำใน server.ts ไม่ทำให้ Bun error ตรงๆ แต่ทำให้ routes หลังจากนั้นไม่ถูก register (404 เงียบๆ) ใช้เวลาหาสาเหตุเพราะไม่มี error message ชัดเจน

## Lessons Learned

1. **WSL tmux ≠ Windows tmux** — เมื่อ maw API ตอบ sessions แต่ local tmux ว่าง → tmux server อยู่ใน WSL เสมอ ใช้ `wsl tmux` แทน
2. **Duplicate imports ใน Bun อาจ silent-fail** — ไม่ crash แต่ routes หลังจุด import ซ้ำอาจไม่ถูก register ตรวจสอบ imports ก่อน debug routes
3. **Gemini CLI --yolo = auto-accept all** — เทียบเท่ากับ Claude ที่ set allowAll permissions แต่ต้องเพิ่ม `gemini` ใน busy detection regex ทุกจุดที่มี `claude|codex|node`

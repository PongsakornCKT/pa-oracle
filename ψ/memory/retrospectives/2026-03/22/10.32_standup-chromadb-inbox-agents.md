## Honest Feedback

**Friction 1: feed-watcher copy ล้มเหลวซ้ำ** — ทุกครั้งที่แก้ feed-watcher.py แล้ว copy ไป Ubuntu ด้วย `cp /mnt/c/...` ไฟล์ที่ copy ไปมักไม่ใช่ version ล่าสุด สาเหตุอาจเป็น file caching ของ WSL2 `/mnt/c` ต้อง verify ด้วย `grep` ทุกครั้งหลัง copy ควรมี deploy script ที่ copy + verify + restart ในคำสั่งเดียว

**Friction 2: Claude Code TUI ไม่สามารถ automate** — bypass permissions prompt ใช้ Ink TUI ที่ไม่ accept standard terminal input (Down arrow, Tab, number keys) ผ่าน tmux send-keys ทำให้ไม่สามารถ switch model โดย script ได้ ต้องทำ manual ผ่าน tmux attach

**Friction 3: Agent model mismatch** — ตั้งใจให้ probe/pixel/patch ใช้ Sonnet 4.6 แต่ตอน first start ลืมระบุ `--model` เลยใช้ Opus default ตอนจะเปลี่ยนก็ทำไม่ได้เพราะ Friction 2 ควรตั้ง model ใน startup script ตั้งแต่แรก (ซึ่งทำแล้วใน start-agents.sh)

## Lessons Learned

1. Claude Code bypass permissions prompt ใช้ Ink TUI ที่ไม่รับ tmux send-keys — ต้อง manual accept หรือใช้ `skipDangerousModePermissionPrompt: true` (ซึ่งต้อง Claude Code version ที่ support)
2. WSL2 `/mnt/c` file copy อาจ stale — ต้อง verify content หลัง copy ทุกครั้ง
3. Feed-watcher Stop detection ต้องเช็ค `stop_reason == "end_turn"` + question markers (`?`, `ไหม`, `ครับ`) จาก JSONL
4. ChromaDB `recall()` ควร search ข้าม collections ทั้งหมด ไม่ใช่แค่ collection เดียว

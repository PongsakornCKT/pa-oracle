# ChromaDB v1.5+ API Changes

**Date**: 2026-03-21
**Source**: rrr: pa-Oracle-v2
**Tags**: chromadb, python, api-change, vector-db

## Lesson

ChromaDB v1.5.5 changed `list_collections()` to return `Collection` objects directly instead of string names. This means:

```python
# OLD (pre-1.5) — returns list of strings
collections = client.list_collections()
for name in collections:
    col = client.get_collection(name)

# NEW (1.5+) — returns list of Collection objects
collections = client.list_collections()
for col in collections:
    print(col.name, col.count())
```

## Context

Hit this during pa Oracle ChromaDB setup. The error was `TypeError: 'Collection' object cannot be converted to 'PyString'` when passing a Collection object to `get_collection()`.

## Takeaway

Always verify library API behavior against actual installed version, not cached knowledge. Breaking changes in fast-moving libraries like ChromaDB are common between major versions.
